# 🔍 NayaPay Website QA Testing Report

## 📌 Project Overview
This project presents a comprehensive QA testing analysis of the NayaPay public website. The goal was to evaluate the platform from a user experience, functionality, accessibility, and usability perspective.

## 🌐 Website Tested
https://www.nayapay.com

## 🧪 Testing Scope
- Manual Testing
- UI/UX Testing
- Accessibility Testing
- Functional Testing

## 🖥️ Test Environment
- Device: Desktop (Windows)
- Browser: Google Chrome

## 👨‍💻 My Role
- Designed and executed test scenarios
- Identified and documented bugs
- Analyzed usability and accessibility issues
- Provided improvement recommendations

## 🐞 Key Findings
- Identified 11 critical and medium-level issues
- UX problems affecting user flow and clarity
- Accessibility issues (contrast, hover states)
- Functional bug related to URL routing behavior

## 📊 Sample Bugs
- Grammar issue affecting brand credibility
- Duplicate CTA causing confusion
- Missing hover states on buttons
- URL change without page navigation (critical issue)

## 📁 Files Included
- Detailed Bug Report (PDF)
- Screenshots of identified issues

## 💡 What I Learned
- Real-world QA testing workflow
- Importance of UX and accessibility in product quality
- How small UI issues impact user trust
- Identifying critical functional flaws in production systems

## 🚀 Conclusion
This project reflects my ability to analyze real-world products, identify critical issues, and provide actionable insights to improve user experience and software quality.
